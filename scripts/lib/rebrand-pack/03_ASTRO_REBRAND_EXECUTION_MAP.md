# ASTRO REBRAND EXECUTION MAP

Implement the rebrand in this order unless the actual repo suggests a better sequence.

## Phase 1 — central brand/config surfaces
Update or create central definitions for:
- site name
- domain
- default title template
- default description
- organization/publisher info
- primary nav labels
- footer data
- global logo references
- default social images if present

## Phase 2 — global UI and layout
Update:
- base layout
- header
- footer
- shared CTA styles
- color tokens / CSS variables / Tailwind theme if present
- favicons and manifest references

## Phase 3 — page-type templates
Update at the template level:
- homepage
- hub pages
- breed page template
- blog/article template
- names page template
- cost calculator template
- category template
- comparison/review templates if present

## Phase 4 — SEO/schema
Update:
- metadata helpers
- title patterns
- OG tags
- Twitter tags
- schema
- canonical logic
- sitemap config
- robots references
- manifest

## Phase 5 — content, trust, and commercial framing
Improve:
- homepage messaging
- hub intros
- about/disclosure/editorial language
- why trust us modules
- comparison page credibility
- softer smarter CTA framing

## Phase 6 — QA
Run:
- build check
- route sanity check
- broken import/assets check
- old-brand reference search
- visual review of key templates
