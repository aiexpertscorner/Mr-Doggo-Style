# INTERNAL LINKING RULES FOR ASTRO

Implement internal linking in scalable Astro-friendly ways.

## Preferred implementation approaches
Evaluate whether the repo should use:
- frontmatter-driven related sections
- taxonomy tags
- componentized related-link blocks
- shared helper functions
- page-type-based link modules
- curated hub lists in data files

Prefer a hybrid model:
- automated where safe
- curated for highest-value hubs

## Link rules by page type

### Breed page
Link to:
- breed hub
- breed cost page
- names page(s)
- training guide
- grooming guide
- food guide
- health guide
- similar breeds
- related products/comparisons

### Cost calculator page
Link to:
- relevant breed pages
- getting-a-dog content
- puppy checklists
- insurance/wellness info if present

### Dog names page
Link to:
- breed pages
- puppy basics
- new-dog preparation guides

### Category page
Link to:
- top evergreen articles
- comparisons
- relevant breed pages
- adjacent category hubs

### Blog/article page
Link to:
- parent hub
- adjacent evergreen guide
- relevant commercial/support page
- related breed/category where appropriate

## Technical rule
Do not inject spammy huge related-link blocks.
Prefer useful, controlled, page-type-aware modules.
